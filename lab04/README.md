# Lab04

Тема: Замыкания и декораторы в Python

Задание:
- Реализовать замыкание, генерирующее последовательность Фибоначчи.
- Написать декоратор для кэширования (мемоизации) и применить его к замыканию.

---

## Краткая идея

- Замыкание (closure) — это внутренняя функция, которая «захватывает» переменные из внешней функции и продолжает к ним доступ даже после её завершения.
- Декоратор — функция, принимающая другую функцию и возвращающая новую функцию с дополнительным поведением. В задаче декоратор используется для кэширования результатов.

- `lab04/lab04_2.py` — содержит замыкание `fibonacci_factory()` и применяет к внутренней функции декоратор из `lab04_1.py`.

---

## Как это работает — простыми словами

1. В `fibonacci_factory()` создаётся словарь `memo = {0:0, 1:1}` — это память для уже вычисленных значений.
2. Внутренняя функция `fib(n)` обращается к `memo`: если значение уже есть — возвращает его, если нет — вычисляет рекурсивно `fib(n-1) + fib(n-2)` и сохраняет в `memo`.
3. Когда `fib` обёрнут декоратором `@cache_decorator`, дополнительно применяется внешний кэш: декоратор хранит свои результаты в собственном словаре `cache`.

Важно: даже без декоратора само замыкание эффективно, так как `memo` уже сохраняет результаты. Декоратор добавляет ещё один слой кэширования.

## Вывод

В ходе работы изучены и закреплены следующие приёмы:
- Механика замыканий: сохранение состояния в `memo` внутри внешней функции.
