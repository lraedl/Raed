# Lab04

Тема: Замыкания и декораторы в Python

Задание:
- Реализовать замыкание, генерирующее последовательность Фибоначчи.
- Написать декоратор для кэширования (мемоизации) и показать его работу отдельно и в сочетании с замыканием.

---

## Краткая идея

- Замыкание (closure) — внутренняя функция, которая «захватывает» переменные из внешней функции и продолжает к ним доступ даже после её завершения.
- Декоратор — функция, принимающая другую функцию и возвращающая новую функцию с дополнительным поведением. В этой работе декоратор применяется для кэширования результатов.

---

## Файлы в папке `lab04`

- `decor.py`  — файл с реализациями декоратора `cache_decorator` и простыми примерами его работы (квадрат, факториал).
- `zamik.py`  — файл с демонстрацией замыканий (счётчик, приветствие, умножитель) и отдельной секцией с применением декоратора к замыканию (числа Фибоначчи).
- `zamdek.py` — файл, где показан совместный пример: декоратор + замыкание для вычисления чисел Фибоначчи.
- `png4/`     — папка для картинок/скриншотов результатов.

---

## Как запускать (примеры)

1) Пример работы **только декоратора**:

```bash
python3 lab04/decor.py
```
Вы увидите, что вычисления не повторяются при повторных вызовах из-за кэша.

2) Пример работы **только замыканий**:

```bash
python3 lab04/zamik.py
```
Секция замыканий покажет счётчик, разные приветствия и умножители.

3) Пример **декоратор + замыкание** (Фибоначчи):

```bash
python3 lab04/zamdek.py
```
Здесь внутренняя функция `fib` захватывает `memo`, а декоратор добавляет внешний кэш — оба механизма вместе дают быстрые вычисления.

---

## Описание работы (коротко)

1. В `zamik.py` и `zamdek.py` используется `memo` — словарь, который хранит уже вычисленные значения Фибоначчи. Это и есть мемоизация внутри замыкания.
2. В `decor.py` декоратор `cache_decorator` поддерживает отдельный словарь `cache` и оборачивает функцию, чтобы возвращать результаты из кэша при повторных вызовах.
3. Комбинация (`zamdek.py`): замыкание даёт локальный кэш `memo`, декоратор даёт внешний кэш `cache` — вместе они работают корректно и быстро.
